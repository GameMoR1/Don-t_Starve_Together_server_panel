## Первый вход

1. Откройте в браузере `http://<IP-сервера>:8000/`
2. Войдите: **admin** / **admin123**
3. Перейдите в **Настройки** → смените пароль (минимум 8 символов, заглавная, цифра, спецсимвол)
4. Рекомендуется включить **двухфакторную аутентификацию (2FA)**

### Роли пользователей

| Роль | Что может |
|------|-----------|
| **viewer** | Просмотр статуса, логов, игроков |
| **operator** | Запуск/остановка сервера, бэкапы, выбор мира |
| **admin** | Редактирование конфигов, токена, списков, модов |
| **owner** | Всё + управление пользователями, установка DST |

## Обзор интерфейса

### Обзор (Dashboard)

Главная страница: статус Master и Caves, число игроков онлайн, системные метрики (CPU, RAM, диск), краткая статистика по игрокам. Данные **обновляются автоматически** каждые ~20 секунд.

### Запуск (Setup)

Мастер первичной настройки: проверка готовности, Cluster Token, пресеты сервера, подсказки по подключению.

### Сервер

Управление кластером, выбор мира, пересборка мира, установка/обновление DST. Статус шардов обновляется каждые ~5 секунд.

### Конфиг

Редактирование `cluster.ini`, `server.ini` (Master/Caves), модов, списков игроков, Cluster Token.

### Игроки

Онлайн, история сессий, время в игре, график активности. Автообновление каждые ~12 секунд.

### Логи

Просмотр `server_log.txt` и chat-лога по шардам.

### Бэкапы

Создание, скачивание, восстановление и удаление архивов.

### Пользователи / Аудит / Настройки

Управление аккаунтами, журнал действий, смена пароля и 2FA.

## Запуск сервера с нуля

### Шаг 1. Установка панели и DST

На сервере (от root):

```bash
cd /opt/dst-panel
sudo bash scripts/install.sh
```

Скрипт создаёт пользователя `dstpanel`, ставит SteamCMD, скачивает DST, настраивает systemd и запускает панель.

### Шаг 2. Первичная настройка в панели

1. Откройте **Запуск**
2. Проверьте чеклист готовности (все обязательные пункты должны быть зелёными)
3. Выберите режим:
   - **Онлайн** — сервер виден в браузере игр Klei, нужен Cluster Token
   - **Для друзей** — по паролю, без публикации в списке
   - **Офлайн-кластер** — LAN без токена (включается в `cluster.ini`)

### Шаг 3. Cluster Token (для онлайн-сервера)

1. Получите токен: [accounts.klei.com → Game Servers](https://accounts.klei.com/account/game/servers)
2. В панели: **Запуск** или **Конфиг → Токен** → вставьте токен → Сохранить
3. Токен **не отображается** после сохранения — это нормально

Файл на диске: `~/dst/DoNotStarveTogether/cluster/cluster_token.txt` (права `600`).

### Шаг 4. Выбор мира и старт

1. **Сервер → Миры** — выберите режим запуска (текущий сейв / из библиотеки / новый мир)
2. Нажмите **Применить выбор**
3. Убедитесь, что чеклист «Готов к запуску» зелёный
4. **Запустить кластер** (Master + Caves)

Первый запуск может занять несколько минут — генерируется мир.

### Шаг 5. Подключение из игры

В панели на **Запуск** или в **Конфиг → Подключение** указаны IP, имя кластера и порты. Игроки подключаются через «Присоединиться» → указать IP или поиск по имени (для онлайн-режима).

## Управление кластером

### Кнопки на странице «Сервер»

| Действие | Описание |
|----------|----------|
| **Запустить кластер** | Стартует Master, затем Caves |
| **Остановить кластер** | Корректно останавливает оба шарда |
| **Перезапустить кластер** | Stop → Start; полезно после смены конфига |
| **Запуск / Стоп / Рестарт** (отдельно) | Управление одним шардом |

### Пересборка мира

Удаляет текущие сейвы и создаёт новый мир при следующем запуске. **Необратимо** без бэкапа.

1. Остановите кластер
2. **Сервер → Пересборка мира** → подтвердите
3. Дождитесь завершения (прогресс-бар)
4. Запустите кластер

### Установка и обновление DST

- **Установить DST** — через SteamCMD (если файлы отсутствуют)
- **Принудительное обновление** — обновляет игровые файлы; перед этим лучше сделать бэкап и остановить сервер

Из командной строки:

```bash
sudo UPDATE_DST=1 bash /opt/dst-panel/scripts/install.sh
```

### Перезапуск панели vs сервера

`systemctl restart dst-panel` **не убивает** процессы DST — шарды продолжают работать. Это сделано специально.

## Миры и библиотека сейвов

Библиотека позволяет хранить несколько именованных миров и переключаться между ними перед запуском.

### Режимы запуска

| Режим | Поведение |
|-------|-----------|
| **Текущий на диске** | Запускает то, что уже лежит в `cluster/Master` и `cluster/Caves` |
| **Из библиотеки** | Копирует выбранный слот из библиотеки на диск перед стартом |
| **Новый мир** | Очищает сейвы и создаёт чистый мир |

### Типичный сценарий

1. Играете на мире «Лето 2025»
2. **Импорт текущего сейва** → сохраняете в библиотеку под именем
3. Создаёте новый слот «Зима 2025» или переключаетесь на старый
4. **Применить выбор** → **Запустить кластер**

### Захват текущего мира в библиотеку

Кнопка **Захватить** у существующего слота перезаписывает его текущими файлами с диска (кластер должен быть остановлен).

Хранилище: `/var/lib/dst-panel/world-library/`

## Конфигурация

Раздел **Конфиг** содержит вкладки:

### Кластер (`cluster.ini`)

Основные параметры: слоты игроков, PvP, режим игры, пароль, голосование, **шарды** (`shard_enabled`), порты Master/Caves.

> Панель использует ключ `shard_enabled` (не `enabled`) — это формат Klei.

### Шарды (`server.ini`)

Отдельно для Master и Caves: размер мира, ресурсы, сезоны, боссы и т.д.

После изменений обычно нужен **перезапуск кластера**.

### Списки

- **adminlist.txt** — администраторы (Klei ID `KU_...`)
- **blocklist.txt** — бан
- **whitelist.txt** — белый список (если включён в cluster.ini)

Добавление из раздела **Игроки** или **Конфиг → Списки**.

### Токен

Управление Cluster Token (см. выше).

### Подключение

Сводка: IP, порты, статус токена, офлайн-режим.

## Моды

Раздел **Моды** (или **Конфиг → Моды**):

1. Добавьте Workshop ID или импортируйте коллекцию Steam
2. Включите/выключите моды
3. Сохраните `modoverrides.lua`
4. Перезапустите кластер

Для **названий модов** и импорта коллекций добавьте в `.env`:

```
STEAM_WEB_API_KEY=ваш_ключ
```

Ключ: [steamcommunity.com/dev/apikey](https://steamcommunity.com/dev/apikey)

После смены `.env`: `sudo systemctl restart dst-panel`

## Игроки и статистика

Панель читает логи DST и ведёт учёт сессий.

### Как считается время

| Событие | Источник |
|---------|----------|
| Вход в игру | `[Join Announcement]` в chat-логе |
| Выход | `[Leave Announcement]` или `SendUserDisconnect` в server_log |

Время сессии = разница между join и leave по меткам времени в логах.

### Что смотреть

- **Онлайн сейчас** — активные сессии
- **Все игроки** — суммарное время, число заходов
- **Клик по игроку** — сессии, IP, график по дням

Если статистика пустая — убедитесь, что Master запущен и файлы `server_log.txt` / `server_chat_log.txt` существуют в каталогах шардов.

## Бэкапы

### Из панели

**Бэкапы → Создать** — архив включает:

- конфиги кластера и шардов
- сейвы Master и Caves
- базу панели (пользователи, статистика игроков)
- библиотеку миров
- резервную копию Cluster Token

Перед созданием бэкапа шарды **автоматически останавливаются**.

### Восстановление

1. **Бэкапы → Восстановить** у нужного архива
2. Дождитесь завершения
3. Проверьте мир и токен
4. Запустите кластер

При восстановлении на **другом сервере** после `install.sh` загрузите архив через **Загрузить** и восстановите.

### Автоматические бэкапы

```bash
sudo bash /opt/dst-panel/scripts/backup.sh
```

Архивы: `/var/lib/dst-panel/backups/`, хранение 30 дней (настраивается в скрипте).

## Логи и диагностика

### Логи в панели

**Логи** → выберите шард (Master/Caves) и файл. Удобно при ошибках запуска.

### Логи на сервере

| Путь | Содержимое |
|------|------------|
| `journalctl -u dst-panel` | Панель |
| `.../cluster/Master/server_log.txt` | Master DST |
| `.../cluster/Caves/server_log.txt` | Caves DST |
| `/var/lib/dst-panel/shard-logs/` | Логи запуска шардов панелью |

### Проверка здоровья

```bash
curl -s http://127.0.0.1:8000/api/health
sudo systemctl status dst-panel
```

### Caves не подключается к Master

1. Убедитесь, что `shard_enabled=true` в cluster.ini
2. **Конфиг → Привязка** — проверьте связку шардов
3. **Перезапустить кластер**, подождите 1–2 минуты
4. Смотрите лог Caves на строки про подключение к Master

## Пользователи и безопасность

### Создание пользователей

**Пользователи** (только owner) → добавить логин, пароль, роль.

### 2FA

**Настройки → Включить 2FA** → отсканируйте QR в приложении (Google Authenticator, Aegis и т.п.).

### Аудит

**Аудит** — журнал входов и важных действий (смена конфига, бэкапы, запуск сервера).

### Рекомендации

- Смените пароль `admin` сразу после установки
- Ограничьте порт `8000` firewall (доступ только с вашего IP / VPN)
- Не публикуйте Cluster Token и `.env` в открытый доступ
- Регулярные бэкапы перед обновлением DST и сменой мира

## Частые проблемы

### Сервер не появляется в списке Klei

- Проверьте Cluster Token
- Убедитесь, что офлайн-режим **выключен**
- Открыты UDP-порты на firewall
- Подождите несколько минут после старта

### «Сервер не готов к запуску»

Откройте **Запуск** или **Сервер** — красные пункты чеклиста подскажут, чего не хватает (токен, конфиг, файлы DST, мир).

### Панель недоступна после обновления

```bash
sudo journalctl -u dst-panel -n 50 --no-pager
sudo bash /opt/dst-panel/scripts/install.sh
```

### Игроки есть онлайн, но время = 0

Проверьте наличие `[Join Announcement]` в `server_chat_log.txt`. После обновления панели статистика пересчитывается из логов автоматически (однократно).

### DST упал после restart панели

Не должно происходить. Если произошло — проверьте версию `dst-panel.service` (`KillMode=process`) и переустановите:

```bash
sudo bash /opt/dst-panel/scripts/install.sh
```

### Нужна помощь

Соберите:

1. Скрин или текст ошибки из панели
2. `journalctl -u dst-panel -n 100`
3. Последние 50 строк `server_log.txt` проблемного шарда
